# Bauer Copy Edits - Agent Implementation Task

## Objective

Apply all suggested copy edits from `bauer-parse-result.json` to target file(s) specified in document metadata.

## Input Data Structure

```typescript
{
  metadata: {
    document_title: string
    document_id: string
    extraction_time: ISO8601
    extraction_duration: nanoseconds
  }
  summary: {
    total_suggestions: number
    total_files: number
    by_file: { [filepath]: { insertions, deletions, replacements } }
    by_type: { insert, delete, replace: number }
  }
  document_metadata: {
    suggested_url: string  // Primary target file
    page_title: string
    page_description: string
    raw: { [key]: string }
  }
  file_mappings: {
    [filepath]: {
      suggested_file: string
      source_reference: string
      suggestion_count: number
      suggestion_ids: string[]
    }
  }
  actionable_suggestions: [
    {
      id: string
      file: string
      anchor: {
        preceding_text: string
        following_text: string
      }
      change: {
        type: "insert" | "delete" | "replace"
        new_text?: string
        original_text?: string
      }
      verification: {
        text_before_change: string
        text_after_change: string
      }
    }
  ]
}
```

## Algorithm

### Phase 1: Parse Input
- [ ] Load `bauer-parse-result.json`
- [ ] Extract `summary.by_file` to identify all target files
- [ ] Extract `metadata.document_title` for commit message
- [ ] Count total suggestions: `summary.total_suggestions`

### Phase 2: For Each Target File
For each file in `file_mappings`:
  1. Load/read the target file content
  2. For each suggestion where `suggestion.file == target_file`:
     - Execute `applySuggestion(suggestion, file_content)`
     - Store modified content
  3. Write modified content back to file
  4. Track success/failure

### Phase 3: Apply Suggestion Algorithm

```pseudocode
function applySuggestion(suggestion, file_content):
  anchor = suggestion.anchor
  change = suggestion.change
  
  // Construct the search pattern
  search_text = anchor.preceding_text + change.original_text + anchor.following_text
  
  // Locate in file
  position = file_content.find(search_text)
  if position == NOT_FOUND:
    log("ERROR: Could not locate anchor for suggestion " + suggestion.id)
    return false
  
  // Apply change based on type
  switch change.type:
    case "insert":
      replacement = anchor.preceding_text + change.new_text + change.original_text + anchor.following_text
    case "delete":
      replacement = anchor.preceding_text + anchor.following_text
    case "replace":
      replacement = anchor.preceding_text + change.new_text + anchor.following_text
  
  // Perform replacement
  file_content = file_content.replace(search_text, replacement)
  
  // Optional: Verify against expected result
  if replacement != change.verification.text_after_change:
    log("WARNING: Verification mismatch for " + suggestion.id)
    // Decision: Continue or abort (recommend continue with warning)
  
  return true
```

### Phase 4: Validation

For a sample of 5-10 suggestions (randomly or first N):
- [ ] Verify the change is in place
- [ ] Verify `verification.text_after_change` matches actual text
- [ ] Log any mismatches

### Phase 5: Commit & Push

```bash
git add -A
git commit -m "Apply {total_suggestions} copy edits from bauer-parse-result.json

Summary:
- {insertions} insertions
- {deletions} deletions  
- {replacements} replacements

Target files:
{list of file_mappings with change counts}

Document: {metadata.document_title}
Document ID: {metadata.document_id}"

git push
```

## Key Fields Reference

| Field | Purpose |
|-------|---------|
| `suggestion.id` | Unique identifier (for debugging) |
| `suggestion.file` | Target file path (use this to know which file to modify) |
| `suggestion.anchor.preceding_text` | Text BEFORE the change (for locating) |
| `suggestion.anchor.following_text` | Text AFTER the change (for locating) |
| `suggestion.change.type` | Operation: insert, delete, or replace |
| `suggestion.change.new_text` | Text to insert (insert/replace only) |
| `suggestion.change.original_text` | Text to remove (delete/replace only) |
| `suggestion.verification.*` | Before/after strings (for validation) |

## Error Handling

**Critical Errors** (stop processing):
- Cannot load `bauer-parse-result.json`
- Cannot read target file from disk
- Cannot write modified file to disk
- Invalid JSON structure

**Non-Critical Errors** (log and continue):
- Anchor text not found (skip this suggestion, log ID)
- Verification mismatch (log ID and continue)
- Duplicate suggestion IDs (apply both)

## Success Criteria

- [ ] All target files loaded successfully
- [ ] All suggestions applied without critical errors
- [ ] 95%+ of suggestions applied (non-critical errors allowed)
- [ ] Changes committed with appropriate message
- [ ] Branch pushed to origin
- [ ] PR shows all commits

All changes automatically appear in this PR. No additional action needed after push.
